<h2 id="프로젝트-개요">🔻Kindly_Project </h2>

Kindly는 예약 관리 기능에 운영 정책과 데이터 분석 기능을 결합한 서비스입니다.  
단순한 기능 구현을 넘어, 실제 매장 운영의 골칫거리인 **노쇼와 예약 관리 문제를 기술적으로 해결**하는 것을 목표로 개발했습니다. <br>

단순히 예약을 받는 시스템을 넘어 고객이 늦을 때 느끼는 심리적 부담을 줄이고, 사장님이 예약 이행 여부를 확인하기 위해 직접 전화를 해야하는 번거로움을 '지연 알림 버튼' 하나로 해결하는
**‘다정한 기술’**을 구현하고자 했습니다.

또한 노쇼율, 재방문율, 인기 시술 등의 데이터를 분석하여  
단순 관리 도구가 아닌 Happy-Hour 설정 및 타겟 마케팅 등 실질적인 매출 증대를 기대할수있는  <br>
**운영 인사이트를 제공하는 서비스**로 확장했습니다.

<h2 id="개발-환경-및-기술-스택">🔻 개발 환경 및 기술 스택</h2>
📅 개발 기간: 2026.04.10 ~ 2026.04.16 (총 6일) <br>

<br>

| 분류 | 기술 스택 | 상세 내용 |

<h2 id="주요-기능">🔻 주요 기능</h2>

👤 사용자 (User)
- **예약**: 날짜 및 시간 선택 시 중복 예약 방지 및 현재 시간 이후 예약만 허용하는 유효성 검증 로직 구현  
- **예약 관리**: 예약 생성, 조회, 변경, 취소까지 전체 예약 흐름 관리 기능 구현  
- **지연 알림 기능**:  예약 지연 시 간편하게 관리자에게 알림을 전송할 수 있는 기능 제공
  
🛠 관리자 (Admin)
- **권한 관리**: 관리자 전용 로그인 및 ROLE 기반 접근 제어 구현  
- **예약 관리 대시보드**: 전체 예약 현황 조회 및 방문 완료 / 노쇼 / 지연 상태 관리 기능 제공 
- **고객 CRM**:  고객별 예약 이력 및 방문 패턴 조회, 노쇼 횟수 및 지연 알림 이력 확인  

📊 데이터 분석 (Analytics)
- **통계 시각화**: 노쇼율 추이, 재방문율, 인기 시술 TOP 분석 등 핵심 운영 지표 대시보드 제공
- **예약 데이터 분석**: 시간대별 예약 분포 분석을 통해 효율적인 인력 및 장비 배치 인사이트 제공

<h2 id="트러블-슈팅">🔻 트러블 슈팅 & 개발 로그</h2>

1. 백엔드: 데이터 무결성 및 보안
- LocalDateTime과 Oracle DATE 타입 간 매핑 불일치 문제를 해결하여 예약 시간 데이터의 정확도를 확보  
- 동일 고객 당일 중복 예약 차단 및 마지막 SUPER_ADMIN 삭제 금지 정책을 구현하여 시스템 안정성 강화  

<br>

2. 프론트엔드: 사용자 경험(UX) 최적화
- 예약 가능/불가 시간을 명확히 구분하기 위해 커스텀 캘린더를 직접 구현하여 UX 개선  
- 예약 변경 시 지연 알림 상태가 초기화되도록 처리하여 프론트-백엔드 데이터 정합성 유지  

<br>

3. 데이터 분석의 목적
- 단순 수치 제공이 아닌, 인기 시술과 시간대별 예약 데이터를 결합하여 운영 인사이트 도출  
- 예약 패턴 분석을 통해 특정 시간대 집중 현상을 파악하고 운영 개선 방향 제시
